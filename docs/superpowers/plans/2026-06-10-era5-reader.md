# ERA5 Reader (`type: era5`) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a dedicated `type: era5` gridded reader that opens ECMWF ERA5 NetCDF (single-level and pressure-level), standardizes coordinates to `(time, z, lat, lon)`, resolves ERA5's `z`-geopotential naming collision, drops CDS singleton `number`/`expver` dims, and is validated through the pipeline.

**Architecture:** A standalone `ERA5Reader` in `davinci_monet/models/era5.py`, mirroring `models/merra2.py` and reusing `io/reader_utils.py` helpers. Unlike MERRA-2 (whose only quirk was `lev`→`z`), ERA5 needs real normalization that the `generic` reader gets wrong, so the dedicated reader is well-justified.

**Tech Stack:** Python 3.11+, xarray (netCDF4 engine), pytest. NetCDF only (no GRIB/cfgrib). No cdsapi dependency (the reader reads local files; staging is the separate ERA5 downloader plan).

**Scope:** The *reader* half of spec Phase 3 (`docs/superpowers/specs/2026-06-10-reanalysis-sources-design.md`). The ERA5 *downloader* (cdsapi/CDS) and CAMS are separate plans.

---

## Audit Findings (why a dedicated reader, and the design)

1. **Vertical dim must be `z`.** GRID surface extraction triggers on `"z" in model.dims` (`pairing/strategies/grid.py:88`); `_extract_surface` auto-detects from `["lev","z","level",...]` (`pairing/strategies/base.py`); profile pairing hardcodes `level_coord="z"`. So ERA5's pressure-level dim (`pressure_level` in current CDS NetCDF, `level` in older) **must be renamed to `z`**.
2. **The `z` collision.** ERA5 names *geopotential* `z` (units m²/s²). Renaming the level dim to `z` while a `z` data variable exists makes `xarray.rename` raise. **Resolution:** rename the geopotential variable `z`→`geopotential` *before* renaming the level dim to `z`.
3. **`generic` mishandles modern ERA5.** Its `COMMON_COORDINATE_ALIASES` maps `latitude/longitude`→`lat/lon` but NOT `valid_time`→`time` nor `pressure_level`→`z`; and it would crash on the `level`→`z` collision. So a dedicated reader adds genuine value (vs. MERRA-2, where `generic` already worked).
4. **CDS singleton dims.** Current CDS ERA5 NetCDF carries `number` (ensemble member, size 1) and `expver` (size 1, or merged). These must be squeezed/dropped so the dataset is `(time, [z,] lat, lon)`.
5. **No cdsapi / no ERA5 staged.** Reader has no cdsapi import; the real-data smoke test is `skipif`-gated and will SKIP until ERA5 is staged via the downloader plan (or manually).

---

## Open Choices (confirm at plan review)

1. **Geopotential rename target** — `z` → **`geopotential`** (proposed; CF standard_name is `geopotential`). Configs referencing ERA5 geopotential then use `geopotential`, not `z`. Alternative: `zg` (geopotential height) — but ERA5 `z` is geopotential, not height, so `geopotential` is more correct.
2. **Longitude normalization** — ERA5 defaults to **0–360°E**; obs/other sources often use −180–180. Proposed: **normalize lon to −180–180 and sort** in the reader (Task 4) so GRID pairing aligns. This reindexes data. Alternative: leave 0–360 and handle at pairing. Recommended: normalize (it prevents a subtle cross-source misalignment bug). Task 4 is self-contained and can be dropped if you prefer leaving it.
3. **Latitude order** — ERA5 is N→S (90→−90, descending). Proposed: **leave as-is** (auto-detect handles surface; renderers/regrid accept descending lat). Not reordered.

```python
"""ERA5 reader.

Reads ECMWF ERA5 NetCDF (single-level and pressure-level) as a GRID source.

ERA5 quirks handled here (the ``generic`` reader does not):

* Geopotential is named ``z`` — the same name as our canonical vertical
  dimension. The reader renames the geopotential *variable* to
  ``geopotential`` before renaming the pressure-level dim to ``z``.
* The vertical dim (``pressure_level`` in current CDS NetCDF, ``level`` in
  older files) is renamed to ``z`` so downstream GRID surface extraction
  (which keys on ``"z" in dims``) works.
* ``valid_time`` is renamed to ``time``; CDS singleton ``number``/``expver``
  dims are squeezed/dropped.
* Longitude is normalized from 0-360 to -180-180 (sorted) so GRID pairing
  aligns with other sources.

Surface extraction stays single-sourced downstream; the reader only
preserves ``z`` ordering. ERA5 pressure levels are stored ascending
(1 hPa ... 1000 hPa) so the surface (max pressure) is the LAST index, which
``surface_level_index`` auto-detects (values increase with index).
"""

from __future__ import annotations

from pathlib import Path
from typing import Any, Sequence

import xarray as xr

from davinci_monet.core.protocols import DataGeometry
from davinci_monet.core.registry import source_registry
from davinci_monet.io.reader_utils import (
    retry_transient_open,
    select_variables,
    set_geometry_attr,
    standardize_dims,
    validate_file_list,
)


@source_registry.register("era5")
class ERA5Reader:
    """Reader for ECMWF ERA5 NetCDF output."""

    @property
    def name(self) -> str:
        """Return reader name."""
        return "era5"

    @property
    def geometry(self) -> DataGeometry:
        """ERA5 output is gridded."""
        return DataGeometry.GRID
```

```python
    # Singleton CDS dims to squeeze/drop so the result is (time,[z,]lat,lon).
    _DROP_DIMS = ("number", "expver")

    def open(
        self,
        file_paths: Sequence[str | Path],
        variables: Sequence[str] | None = None,
        **kwargs: Any,
    ) -> xr.Dataset:
        """Open ERA5 NetCDF files and standardize to (time, z, lat, lon).

        Resource-fork ``._*`` sidecars are ignored.
        """
        real = [Path(f) for f in file_paths if not Path(f).name.startswith("._")]
        file_list = validate_file_list(real, source_label="ERA5")

        def _open() -> xr.Dataset:
            if len(file_list) > 1:
                ds = xr.open_mfdataset(
                    [str(f) for f in file_list],
                    combine="by_coords",
                    parallel=True,
                    **kwargs,
                )
            else:
                ds = xr.open_dataset(str(file_list[0]), **kwargs)
            return select_variables(ds, variables)

        ds = retry_transient_open(_open, context="Opening ERA5 files")
        return self._standardize_dataset(ds)

    def _standardize_dataset(self, ds: xr.Dataset) -> xr.Dataset:
        """Resolve ERA5 naming quirks and standardize to (time, z, lat, lon)."""
        # 1. Geopotential variable 'z' collides with the canonical vertical
        #    dim 'z'; rename the variable before renaming the level dim.
        if "z" in ds.data_vars:
            ds = ds.rename({"z": "geopotential"})

        # 2. Standardize dim/coord names. Only present dims are renamed.
        ds = standardize_dims(
            ds,
            {
                "valid_time": "time",
                "latitude": "lat",
                "longitude": "lon",
                "pressure_level": "z",
                "level": "z",
                "isobaricInhPa": "z",
            },
        )

        # 3. Drop CDS singleton dims (ensemble member / experiment version).
        for dim in self._DROP_DIMS:
            if dim in ds.dims and ds.sizes[dim] == 1:
                ds = ds.squeeze(dim, drop=True)
            elif dim in ds.coords and dim not in ds.dims:
                ds = ds.drop_vars(dim)

        return set_geometry_attr(ds, DataGeometry.GRID)
```

```python
def _make_pressure_level(nt: int = 2) -> xr.Dataset:
    """An ERA5-like pressure-level dataset including geopotential named 'z'.

    Pressure levels ascending (1 ... 1000 hPa) per ERA5 NetCDF, so the
    surface (1000 hPa) is the LAST index.
    """
    valid_time = np.array(
        ["2026-04-01T00", "2026-04-01T06"], dtype="datetime64[ns]"
    )[:nt]
    latitude = np.linspace(90.0, -90.0, 5)
    longitude = np.linspace(0.0, 359.0, 6)
    plev = np.array([1.0, 100.0, 500.0, 850.0, 1000.0])  # ascending hPa
    nz = len(plev)
    rng = np.random.default_rng(1)
    shape = (1, nt, nz, 5, 6)
    t = rng.uniform(200.0, 300.0, size=shape).astype(np.float32)
    geo = rng.uniform(0.0, 5e5, size=shape).astype(np.float32)
    dims = ("number", "valid_time", "pressure_level", "latitude", "longitude")
    return xr.Dataset(
        {"t": (dims, t), "z": (dims, geo)},  # 'z' = geopotential (collision!)
        coords={
            "number": [0],
            "valid_time": valid_time,
            "pressure_level": plev,

            "longitude": longitude,
        },
    )
```

```python
def _obs_grid(varname: str, path: Path, seed: int) -> None:
    times = np.array(
        ["2026-04-01T00", "2026-04-01T06", "2026-04-01T12"],
        dtype="datetime64[ns]",
    )
    rng = np.random.default_rng(seed)

def _run(tmp_path: Path, era5_file: Path, model_var: str) -> None:
    obs_dir = tmp_path / "obs"
    obs_dir.mkdir()
    _obs_grid("t2m_obs", obs_dir / "obs.nc", seed=2)
    out_dir = tmp_path / "out"
    config = {
        "analysis": {
            "start_time": "2026-04-01",
            "end_time": "2026-04-01",
            "output_dir": str(out_dir),
            "log_dir": str(tmp_path / "logs"),
        },
        "sources": {
            "era5": {
                "type": "era5",
                "role": "model",
                "files": str(era5_file),
                "variables": {model_var: {"units": "K"}},
            },
            "ref": {
                "type": "generic",
                "role": "obs",
                "files": str(obs_dir / "*.nc"),
                "variables": {"t2m_obs": {"units": "K"}},
            },
        },
        "pairs": {
            "era5_vs_ref": {
                "sources": ["era5", "ref"],
                "reference": "ref",
                "variables": {"era5": model_var, "ref": "t2m_obs"},
            }
        },
        "plots": {
            "sc": {"type": "scatter", "pairs": ["era5_vs_ref"], "title": "T2M"},
        },
        "stats": {"output_table": True, "metrics": ["N", "MB", "RMSE", "R"]},
    }
    cfg = tmp_path / "config.yaml"
    cfg.write_text(yaml.safe_dump(config))
    result = PipelineRunner(show_progress=False).run_from_config(str(cfg))
    failed = [
        f"{s.stage_name}: {s.error}"
        for s in result.stage_results
        if s.status.name == "FAILED"
    ]
    assert result.success, f"Pipeline failed: {failed}"
    assert list(out_dir.rglob("*.csv")), "expected a stats CSV"


def test_era5_single_level_pipeline(tmp_path: Path) -> None:
```
